### 问题痛点

脚本受限：用代码访问网页没有登录状态，并且受反爬策略影响严重。

操作精度差：Agent 原生能力通过图像识别操作用户浏览器，操作精度差。

学习能力弱：每次都要重新理解页面，无法记住网站结构

时间效率低：**每一步**都需“看页面→分析→调工具”，比人慢10~20倍

### 解决方案

Braiser 的核心思路是：把“浏览器操作”从一次性的 Agent 对话，升级为一个可学习、可校验、可复用的自动化系统。

它不是单纯让 Agent 控制浏览器，而是把网页任务拆成三层：环境层解决“能不能进网页”，记忆层解决“是否每次都要重新理解”，执行层解决“如何快速、稳定地完成操作”。

#### 环境层：接入用户真实浏览器

Agent 不再绕过用户浏览器去单独访问网站，而是在用户已有浏览器环境中工作。这样可以天然继承登录状态、权限上下文和真实页面状态，解决传统脚本在登录、鉴权、反爬场景下难以运行的问题。

#### 记忆层：把网站变成可复用知识

系统会把访问过的页面、页面功能和任务经验沉淀为本地网站记忆。下次面对同一网站或相似任务时，Agent 不需要重新从零观察和理解，而是可以基于已有的网站知识进行规划。

#### 执行层：规划交给 Agent，动作交给脚本

Braiser 让 Agent 负责理解任务和生成操作计划，把具体网页动作交给自动化脚本执行。执行过程中系统持续校验页面状态，正常情况快速推进，异常情况再交回 Agent 分析处理，从而兼顾灵活性和效率。

### 产品功能

#### 自然语言创建网页任务

用户直接用自然语言描述目标，例如“进入 GitHub 项目页面，下载最新 release 文件”或“打开后台系统，批量导出本周数据”。Braiser 会把任务转化为可执行的网页操作流程。

#### 当前浏览器一键执行

任务可以在用户当前浏览器中执行，不需要用户重新登录，也不需要单独配置账号密码。用户看到的是一个贴近真实使用环境的网页自动化助手。

#### 网站任务记忆库

Braiser 会按网站保存历史任务、常用页面、成功路径和可复用操作。用户可以查看某个网站已经学会了哪些任务，也可以继续让 Agent 扩展新的能力。

#### 云端知识库与脚本模板

Braiser 官方会提供常见网站的基础知识库和自动化脚本模板，例如 GitHub、主流电商后台、SaaS 管理后台中的高频操作。用户可以直接下载使用，也可以在本地根据自己的账号、页面状态和业务需求继续调整。

#### 自动化脚本复用

对于已经成功执行过的高频任务，用户可以直接再次运行保存下来的自动化脚本，无需每次重新让 Agent 推理。这个能力尤其适合批量下载、周期性导出、重复表单处理等任务。

#### 网站自动探索

Braiser 可以让 Agent 自由探索网站用法，并自动保存页面快照、整理页面用法、编写脚本函数。

#### 风险动作确认

当任务涉及支付、删除、提交敏感信息、修改关键配置等高风险动作时，系统暂停自动执行并请求用户确认，避免 Agent 误操作或泄露个人信息。

### 技术亮点

#### 面向 LLM 的网页结构清洗

Braiser 不直接把完整网页源码或截图丢给 LLM，而是提取 DOM、可交互元素、表单、按钮、链接和主要文本，并转换为更适合 LLM 理解的结构化输入或 markdown 表达。这样可以减少无关信息，提高页面理解准确率。

#### 页面结构快照机制

Braiser 会为访问过的页面生成结构快照，记录关键元素、状态和功能区域。脚本执行后，系统可以通过快照对比判断页面是否进入预期状态，而不必每一步都重新调用 LLM 观察整个页面。

#### Playwright 脚本驱动的高速执行

相比“观察一步、思考一步、操作一步”的传统 Agent 模式，Braiser 使用 Playwright 脚本批量执行操作，并在每步操作后自动用页面结构快照进行校验。LLM 主要负责规划和处理异常，具体网页动作交给脚本完成，从而显著降低时间和 token 成本。

#### 操作函数自动抽象

Braiser 会把稳定出现的网页操作抽象为可调用函数，例如批量下载、表单填写、数据导出等。之后 Agent 规划任务时，可以直接调用这些函数，而不是重新生成完整操作序列。

#### 异常检测与 LLM 接管

当页面结构快照与预期不一致时，Braiser 不会盲目继续执行脚本，而是将异常状态交给 LLM 重新分析。这让系统同时具备脚本自动化的速度和 Agent 面对复杂变化时的适应能力。

### 竞品对比

| 类型 | 代表产品 | 优势 | 局限 | Braiser 的差异 |
|------|----------|------|------|----------------|
| 通用浏览器 Agent | OpenAI Operator、ChatGPT Agent | 通用能力强，能完成开放式网页任务 | 偏一次性执行，不擅长长期记住某个网站和复用流程 | Braiser 强调网站记忆和自动化脚本沉淀，越用越熟 |
| 企业 RPA | UiPath、Power Automate、影刀 RPA | 稳定、可审计，适合大型企业流程 | 配置重，学习成本高，小团队和个人使用门槛高 | Braiser 用自然语言和真实浏览器接入降低使用门槛 |
| 通用桌面操作 Agent | Agivar、Anthropic Computer Use、Microsoft Copilot | 覆盖桌面和网页，能力边界更宽 | 场景过泛，网页任务深度和复用能力不足 | Braiser 专注网页场景，把登录态、DOM、脚本复用做深 |
| 浏览器自动化工具 / MCP 插件 | Playwright、浏览器 MCP 工具 | 开发者可控，适合写脚本和调试 | 面向技术用户，缺少面向普通用户的记忆、确认和复用体验 | Braiser 将底层浏览器自动化包装成用户可直接使用的产品 |
