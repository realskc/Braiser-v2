# Braiser v0.1.0

## 目标

v0.1.0 是 Braiser 第一个可运行的本地应用里程碑。它验证三件事：Electron 应用壳、DeepSeek API 调用、本地文件存储。这个版本不连接浏览器，也不执行浏览器自动化。

核心流程：

```text
用户在本地应用输入普通问题
-> 本地应用调用 DeepSeek API
-> 本地应用展示单轮回答
-> 本地应用把这次 QA run 保存到本地
```

## 已实现范围

- Electron + React + Vite + TypeScript 本地应用。
- 设置面板：DeepSeek API key、base URL、model。
- DeepSeek OpenAI-compatible Chat Completions 调用。
- 默认模型：`deepseek-v4-flash`。
- 通过 API 参数关闭 DeepSeek thinking mode：`thinking: { type: "disabled" }`。
- 单轮普通 QA 展示。
- 基础文件存储：`meta.json`、`events.jsonl`、`messages.json`、`answer.txt`。
- Electron 主进程启动日志：`.braiser-electron-user-data/main.log`。

## 不在本版本范围内

- 不接入 Chrome Extension。
- 不读取或操作浏览器页面。
- 不生成 playwright-like script。
- 不执行 AI 生成的脚本。
- 不支持多轮上下文。
- 不实现 Agent Loop、Skill、Snapshot 持久化。

## 运行方式

在仓库根目录运行：

```powershell
npm.cmd install
npm.cmd run dev
```

如果当前 shell 里有 `ELECTRON_RUN_AS_NODE=1`，开发脚本会在启动 Electron 前清掉它。

## 配置

设置文件路径：

```text
braiser-data/settings.json
```

结构：

```json
{
  "deepSeekApiKey": "sk-...",
  "deepSeekBaseUrl": "https://api.deepseek.com",
  "deepSeekModel": "deepseek-v4-flash"
}
```

API key 查找顺序：

```text
DEEPSEEK_API_KEY 环境变量
-> braiser-data/settings.json 中的 deepSeekApiKey
```

`settings.json` 是本地开发状态，随 `braiser-data/` 一起被 `.gitignore` 忽略。

## Run 产物

每次 QA 请求会创建一个 run：

```text
braiser-data/
  settings.json
  runs/
    <run-id>/
      meta.json
      events.jsonl
      messages.json
      answer.txt
```

文件含义：

- `meta.json`：run id、类型、用户问题、状态、时间戳、错误摘要。
- `events.jsonl`：追加写入的状态事件。
- `messages.json`：本次单轮请求消息和 assistant 回答。
- `answer.txt`：最终回答文本。

run 产物里不能写入 DeepSeek API key 或 Authorization header。

## 验收标准

用户输入普通问题后，应用能够：

- 使用配置好的 API key 调用 DeepSeek。
- 展示单轮回答。
- 将 run 产物保存到本地。
- 不把 API key 写入日志或 run 文件。
